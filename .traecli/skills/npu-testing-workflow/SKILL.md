# NPU Testing Workflow Skill

This skill guides agents to systematically complete three core tasks on NPU (Ascend) devices:

1. **Service Deployment Script Development**: Create executable service deployment scripts with complete environment variable configuration, feature switches, parameter tuning, and parallel strategies
2. **Client Benchmarking Script Development**: Develop professional client benchmarking scripts for functional verification, performance testing, and accuracy evaluation
3. **Test Result Summary and Report Generation**: Design automated result collection mechanism and generate structured test reports

## Prerequisites

Before starting, ensure the following documents are reviewed from `./docs/platforms/`:
- `ascend_npu.md` - Installation and basic configuration
- `ascend_npu_best_practice.md` - Best practices for DeepSeek/Qwen models
- `ascend_npu_environment_variables.md` - Environment variables reference
- `ascend_npu_support_features.md` - Supported features and server arguments
- `ascend_npu_support_models.md` - Supported models list
- `ascend_npu_quantization.md` - Quantization support
- `ascend_npu_deepseek_example.md` - DeepSeek model examples
- `ascend_npu_qwen3_examples.md` - Qwen3 model examples

### Model Adaptation Prerequisite

If the target model is **not yet supported** in SGLang:
1. Use the **sglang-model-adapter** skill first to adapt the model
2. Ensure the model passes basic validation before running this testing workflow
3. Reference the model's tutorial doc (generated by adapter) for deployment details

### Input from Model Adapter

When transitioning from `sglang-model-adapter`, expect:
- Model path and configuration
- Working server startup command
- Known limitations or issues
- Basic test results (functional tests passed)

## Task 1: Service Deployment Script Development

### 1.1 Environment Variable Configuration

Create deployment scripts with the following environment variable categories:

#### System Performance Settings
```bash
echo performance | tee /sys/devices/system/cpu/cpu*/cpufreq/scaling_governor
sysctl -w vm.swappiness=0
sysctl -w kernel.numa_balancing=0
sysctl -w kernel.sched_migration_cost_ns=50000
export SGLANG_SET_CPU_AFFINITY=1
```

#### CANN Environment Setup
```bash
source /usr/local/Ascend/ascend-toolkit/set_env.sh
source /usr/local/Ascend/nnal/atb/set_env.sh
export PATH=/usr/local/Ascend/8.5.0/compiler/bishengir/bin:$PATH
```

#### Memory and Stream Configuration
```bash
export PYTORCH_NPU_ALLOC_CONF=expandable_segments:True
export STREAMS_PER_DEVICE=32
```

#### Communication Settings (HCCL)
```bash
export HCCL_BUFFSIZE=1600
export HCCL_SOCKET_IFNAME=lo
export GLOO_SOCKET_IFNAME=lo
export HCCL_OP_EXPANSION_MODE=AIV
```

#### NPU-Specific Optimizations
```bash
export SGLANG_NPU_USE_MLAPO=1
export SGLANG_USE_FIA_NZ=1
export SGLANG_ENABLE_OVERLAP_PLAN_STREAM=1
export SGLANG_ENABLE_SPEC_V2=1
```

#### DeepEP Configuration (for MoE models)
```bash
export DEEP_NORMAL_MODE_USE_INT8_QUANT=1
export SGLANG_DEEPEP_NUM_MAX_DISPATCH_TOKENS_PER_RANK=32
export DEEPEP_NORMAL_LONG_SEQ_PER_ROUND_TOKENS=1024
export DEEPEP_NORMAL_LONG_SEQ_ROUND=16
```

### 1.2 Deployment Mode Selection

#### PD Mixed Mode (Single Instance)
```bash
python -m sglang.launch_server \
    --model-path ${MODEL_PATH} \
    --tp 16 \
    --trust-remote-code \
    --attention-backend ascend \
    --device npu \
    --quantization modelslim \
    --host 127.0.0.1 --port 8000 \
    --mem-fraction-static 0.8 \
    --max-running-requests 256 \
    --cuda-graph-bs 8 16 24 32
```

#### PD Separation Mode (Prefill Server)
```bash
export ASCEND_MF_STORE_URL="tcp://PREFILL_IP:PORT"
python -m sglang.launch_server \
    --model-path ${MODEL_PATH} \
    --disaggregation-mode prefill \
    --host ${PREFILL_IP} \
    --port 8000 \
    --disaggregation-bootstrap-port 8995 \
    --disaggregation-transfer-backend ascend \
    --tp-size 16 \
    --attention-backend ascend \
    --device npu
```

#### PD Separation Mode (Decode Server)
```bash
export ASCEND_MF_STORE_URL="tcp://PREFILL_IP:PORT"
python -m sglang.launch_server \
    --model-path ${MODEL_PATH} \
    --disaggregation-mode decode \
    --host ${DECODE_IP} \
    --port 8001 \
    --tp-size 32 --dp-size 32 \
    --attention-backend ascend \
    --device npu \
    --deepep-mode low_latency
```

#### Router Configuration
```bash
python -m sglang_router.launch_router \
    --pd-disaggregation \
    --policy cache_aware \
    --prefill http://PREFILL_IP:8000 8995 \
    --decode http://DECODE_IP:8001 \
    --host 127.0.0.1 \
    --port 6688
```

### 1.3 Key Server Arguments Reference

| Argument | Description | Typical Values |
|----------|-------------|----------------|
| `--tp-size` | Tensor parallelism size | 1, 2, 4, 8, 16, 32 |
| `--dp-size` | Data parallelism size | 1, 4, 8, 16, 32 |
| `--mem-fraction-static` | Memory fraction for static allocation | 0.6-0.86 |
| `--max-running-requests` | Maximum concurrent requests | 32-832 |
| `--cuda-graph-bs` | CUDA graph batch sizes | 8 16 24 32 |
| `--chunked-prefill-size` | Chunked prefill size | -1 (disabled), 32768 |
| `--max-prefill-tokens` | Maximum prefill tokens | 16384-28680 |
| `--moe-a2a-backend` | MoE all-to-all backend | deepep, ascend_fuseep |
| `--deepep-mode` | DeepEP mode | normal, low_latency, auto |
| `--quantization` | Quantization method | modelslim |

## Task 2: Client Benchmarking Script Development

### 2.1 Performance Testing with bench_serving

```bash
python -m sglang.bench_serving \
    --dataset-name random \
    --backend sglang \
    --host 127.0.0.1 \
    --port 6688 \
    --max-concurrency 256 \
    --random-input-len 3500 \
    --random-output-len 1500 \
    --num-prompts 1024 \
    --random-range-ratio 1
```

#### Key Benchmark Parameters
- `--dataset-name`: random, sharegpt, sonnet, etc.
- `--max-concurrency`: Maximum concurrent requests
- `--random-input-len`: Input token length
- `--random-output-len`: Output token length
- `--num-prompts`: Number of test prompts
- `--request-rate`: Request rate (for throughput testing)

### 2.2 Accuracy Testing with GSM8K

```python
from types import SimpleNamespace
from sglang.test.few_shot_gsm8k import run_eval

def gsm8k_test():
    args = SimpleNamespace(
        num_shots=5,
        data_path=None,
        num_questions=200,
        max_new_tokens=512,
        parallel=32,
        host="http://127.0.0.1",
        port=6688,
    )
    metrics = run_eval(args)
    print(f"Accuracy: {metrics['accuracy']}")
```

### 2.3 Performance Metrics to Collect

| Metric | Description | Measurement Method |
|--------|-------------|-------------------|
| **TTFT** | Time to First Token | From bench_serving output |
| **TPOT** | Time Per Output Token | From bench_serving output |
| **Throughput** | Output tokens per second | From bench_serving output |
| **ITL** | Inter-Token Latency | From bench_serving output |
| **Accuracy** | Model accuracy on benchmarks | From GSM8K/other benchmarks |
| **Memory Usage** | NPU memory utilization | System monitoring |

## Execution Workflow

When invoked, follow this systematic workflow:

### Step 1: Gather Requirements
1. Identify target model and hardware (A2/A3)
2. Determine deployment mode (PD Mixed/Separation)
3. Define performance targets (latency vs throughput)
4. Specify test datasets and accuracy benchmarks

### Step 2: Create Deployment Script
1. Generate environment variable configuration
2. Configure server arguments based on model type
3. Set up parallel strategy (TP/DP/EP)
4. Create startup script for single/multi-node deployment

### Step 3: Develop Benchmark Scripts
1. Create performance test script using `bench_serving`
2. Create accuracy test script (GSM8K or custom)
3. Configure test parameters (concurrency, input/output lengths)
4. Add result collection and parsing logic

### Step 4: Execute Tests
1. Start model service with deployment script
2. Wait for service readiness
3. Run performance benchmarks
4. Run accuracy benchmarks
5. Collect all metrics

### Step 5: Generate Report
1. Compile test environment details
2. Summarize functional verification results
3. Present performance metrics with comparisons
4. Document accuracy evaluation results
5. List issues and optimization suggestions
6. Provide conclusions and next steps

### Step 6: Feedback Loop (If Issues Found)

If testing reveals issues that require code changes:

1. **Document the issue clearly**:
   - Error message or symptom
   - Reproduction steps
   - Expected vs actual behavior

2. **Return to `sglang-model-adapter`** if:
   - Model architecture support is incomplete
   - NPU-specific code has bugs
   - Weight mapping is incorrect

3. **Provide feedback to adapter**:
   - Specific test case that failed
   - Relevant logs and error messages
   - Suggested fix (if known)

## Model-Specific Configurations

### DeepSeek-R1 / DeepSeek-V3
- Use `--quantization modelslim` for W8A8/W4A8 models
- Enable speculative decoding: `--speculative-algorithm NEXTN`
- Configure DeepEP: `--moe-a2a-backend deepep`
- Set `--enable-dp-attention` for better throughput

### Qwen3 Series
- Standard models: Use `--attention-backend ascend`
- MoE models: Configure DeepEP backend
- For speculative decoding: Use `--speculative-algorithm EAGLE3`
- Multimodal: Add `--enable-multimodal --mm-attention-backend ascend_attn`

### Hardware-Specific Notes

#### Atlas 800I A2
- Supports 8 NPU devices per node
- Use `--base-gpu-id` for device selection
- RDMA transfer: Set `ASCEND_MF_TRANSFER_PROTOCOL=device_rdma`

#### Atlas 800I A3
- Supports 16 NPU devices per node
- Higher memory bandwidth
- Better performance for large models

## Common Issues and Solutions

| Issue | Possible Cause | Solution |
|-------|---------------|----------|
| OOM Error | Memory fraction too high | Reduce `--mem-fraction-static` |
| Slow TTFT | Prefill bottleneck | Increase `--max-prefill-tokens` |
| Low Throughput | Suboptimal batch size | Adjust `--cuda-graph-bs` |
| Communication Timeout | HCCL buffer too small | Increase `HCCL_BUFFSIZE` |
| Accuracy Drop | Quantization issue | Check model weights and quantization config |
